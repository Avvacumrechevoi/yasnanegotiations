## 1. ПРОБЛЕМА

Ясна — древнее славянское учение о 12-частной структуре мира — существует только в виде книги (А.В. Лебедев, 700+ пунктов) и устных уроков. Нет ни одного цифрового инструмента, который позволял бы:

- Визуализировать Ясну как интерактивную 12-лучевую звезду
- Создавать новые Ясны для любых предметных областей
- Проверять корректность размещения элементов по механикам
- Изучать внутренние связи (кресты, праны, противостояния, физиологию)
- Хранить и передавать Ясны в цифровом формате

Люди, изучающие Ясну, вынуждены рисовать звёзды на бумаге, держать в голове десятки правил и проверять корректность вручную. Преподаватели тратят значительную часть урока на объяснение структуры вместо содержания.

---

## 2. РЕШЕНИЕ

Веб-приложение «Ясна-Конструктор» — рабочая среда для создания, изучения и верификации Ясен.

Центральный элемент интерфейса — **интерактивная 12-лучевая звезда**, которая занимает основную часть экрана. Пользователь работает с ней в 6 режимах: исследует позиции, визуализирует кресты и треугольники стихий, наблюдает физиологию цикла, изучает противостояния, редактирует содержимое.

## 4. ЯДРО ПРОДУКТА: ДВИЖОК ЯСНЫ

### 4.1. Что вычисляется автоматически

Пользователь вводит только **12 названий** (полочек). Всё остальное — результат вычислений:

| Атрибут | Формула | Источник |
|---------|---------|----------|
| Тип (долгое/короткое) | `index % 2 === 0 ? 'long' : 'short'` | Книга п.91–97 |
| Крест | 0,3,6,9 → Опорный; 1,4,7,10 → Правый; 2,5,8,11 → Левый | Книга п.436–499 |
| Прана | `index % 4`: 0→ШЭ, 1→ФО, 2→ЦИ, 3→ХА | Книга п.544–656 |
| Полная противоположность | `(index + 6) % 12` | Книга п.223 |
| Половинная противоп. (борьба) | Зеркало по оси 3–9 | Книга п.233 |
| Половинная противоп. (единство) | Зеркало по оси 0–6 | Книга п.253 |
| Треугольник стихии | Каждая прана → 3 позиции через 4 | Книга п.679–683 |
| Принадлежность к чаше | 4,5,6,7,8 → Свет; 10,11,0,1,2 → Тьма; 3,9 → Горизонт | Книга п.306 |
| Физиология (свет/тьма) | Кривая нарастания/спада, лёгкие/тяжёлые времена | Книга п.273–277 |
| Дуга тепла | Три дуги ФО→ЦИ→ХА→ШЭ→ФО | Книга п.673–678 |
| Роль в цикле | Вера → Бой → Победа (L→O→R) | Книга п.507–508 |
| Угол на звезде | `270 - index * 30` | Схема PDF |

### 4.2. Верификация (чек-лист из 7 уровней)

При создании новой Ясны система проверяет каждый элемент по 7 критериям:

1. **Тип** — элемент соответствует долгому процессу или короткому перелому?
2. **Прана** — качество элемента совпадает с праной позиции (устойчивость/спад/покой/рост)?
3. **Крест** — роль элемента (основа/результат/подготовка) совпадает с крестом?
4. **Противоположность** — элемент напротив является зеркальной противоположностью?
5. **Соседство** — элемент логически следует из предыдущего и ведёт к следующему?
6. **Половина** — элемент на верхней = явный, на нижней = скрытый?
7. **Языковая связь** — есть аналогия с элементами на той же полочке в других Яснах?

Элемент считается корректно размещённым при прохождении 7 из 7 проверок. Ни одна проверка не может быть пропущена.

### 4.3. Механики визуализации

| Режим | Что показывает | Зачем |
|-------|---------------|-------|
| Изучение | Информация о выбранной позиции | Понять каждый луч отдельно |
| Три креста | Три группы по 4 луча, раскрашенные цветом, с линиями связи | Увидеть ритм «Вера→Бой→Победа» |
| Треугольники стихий | Четыре треугольника (ШЭ/ФО/ЦИ/ХА) с заливкой | Понять распределение стихий |
| Физиология | Яркость луча = количество света, анимированный маркер по кругу | Ощутить живой цикл нарастания/спада |
| Противостояния | Шесть линий через центр, интерактивное выделение пар | Увидеть зеркальность |
| Редактор | Правая панель с 12 полями ввода | Создать/изменить Ясну |

## 7. КАЧЕСТВО ДАННЫХ

### 7.1. Источники

| Источник | Объём | Использование |
|----------|-------|---------------|
| Книга «Ясна» (Лебедев) | 700+ пунктов, 187 000 символов | Все формулы, механики, физиология |
| Рубрикатор полочек (XLSX) | 22 строки × 12 позиций | Шаблоны |
| 16 видео-уроков (транскрипты) | ~140 000 символов | Данные для позиции 7, обоснования размещения |
| «Русский Крест» | 8 000 символов | Философский контекст |
| Схема Ясны базовая (PDF) | 1 изображение | Эталонная визуализация |

### 7.2. Известные ограничения данных

1. **Позиция 7 в Рубрикаторе** отсутствует (дефект форматирования XLSX). Данные восстановлены из уроков для 7 Ясен (Двора, Дома, Кухни, Спальни, Круговорота, Суток, Месяцев). Для остальных 15 позиция 7 оставлена пустой.

2. **Два набора названий** для Ясны Суток: книга использует длинные описательные («Выход Солнца — Проявление Света»), рубрикатор — краткие («Утро»). В модели данных хранятся оба.

3. **Кривая освещённости** — приблизительная. Книга описывает качественное поведение (ускорение/замедление), не количественные значения.

4. **Крест Веры** (Левый) описан в книге менее подробно, чем Опорный и Правый. Автор указывает: «Для изучения Креста Духа сейчас знаний не хватает. О нём речь впереди» (п.453).

## 8. ПОЛЬЗОВАТЕЛЬСКИЕ СЦЕНАРИИ

### Сценарий 1: Ученик изучает Ясну Суток

1. Открывает приложение → Библиотека → выбирает «Ясна Суток»
2. Видит 12-лучевую звезду с подписями
3. Нажимает на луч 3 → всплывает карточка: «Выход Солнца. ХА (Огонь). Опорный крест. Стрелка Весов — Время Истины»
4. Переключает режим «Три креста» → видит, как лучи 0-3-6-9 связаны линиями
5. Переключает «Физиология» → наблюдает анимацию цикла света
6. Переключает «Треугольники» → видит 4 треугольника стихий

### Сценарий 2: Практик создаёт Ясну Переговоров

1. Библиотека → «Создать с нуля»
2. Вводит название: «Ясна Переговоров»
3. Заполняет полочки: 0 = «Непонимание», 1 = «Первый контакт»...
4. Система показывает подсказку: «Позиция 3 (ХА/Огонь/Опорный) — здесь должно быть главное событие, вспышка, ключевой момент»
5. Вводит: 3 = «Предложение. Презентация»
6. Система проверяет по чек-листу: 7/7 ✅
7. Сохраняет и экспортирует в JSON

### Сценарий 3: Преподаватель готовит урок

1. Открывает «Ясна Двора (Животных)»
2. Включает режим «Противостояния»
3. Нажимает пару 0↔6: «Свинья ↔ Человек» — объясняет: «Свинья всеядна и человек всеяден; свинья внизу (грязь), человек наверху (дом)»
4. Включает «Треугольники» → показывает: «Треугольник Воды (1-5-9): Грызуны, Верблюд, Птицы — все связаны с водой/потоком»
5. Экспортирует визуализацию в PNG для презентации
