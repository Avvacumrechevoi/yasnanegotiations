## 1. СИСТЕМНАЯ ОШИБКА: ПОЗИЦИЯ 7 В РУБРИКАТОРЕ

**Факт:** В Рубрикаторе (XLSX) колонка «7 полочка» **отсутствует** во ВСЕХ строках. Заголовок перескакивает с «6 полочка» сразу на «8 полочка». Данные позиции 7 = пустые.

**Но из уроков** позиция 7 заполнена:
- Двора: Лавка хозяина, детская площадка, кошка (Урок 2 животных)
- Дома: Детская комната (Урок 3 дома)
- Спальни: Красный угол, люлька (Урок 1 спальни)
- Зодиак: Лев ♌ (по порядку зодиака)

**Вывод:** Позиция 7 существует, данные берутся из уроков. Рубрикатор имеет дефект форматирования.

**Статус в документах:** Частично учтено. Нужно добавить явное примечание.

## 3. ОШИБКИ В LOGIC_SPEC

### 3.1. Названия Ясны Суток — расхождение с рубрикатором

Рубрикатор использует **другие названия** для некоторых позиций:

| Позиция | В LOGIC_SPEC (из книги) | В Рубрикаторе | Комментарий |
|---------|------------------------|---------------|-------------|
| 1 | Первый свет | Искра | Разные названия одного времени |
| 3 | Выход Солнца | Утро | Разный акцент |
| 4 | Подъём Солнца | Восход | «Восход» = «Подъём» — синонимы |
| 5 | Последняя тьма | Утренний Салют | «Салют» — другая традиция именования |
| 8 | Спуск Солнца | Закат / Вечерняя Заря | Синонимы |
| 9 | Заход Солнца | Запад / Вечер | Синонимы |
| 11 | Последний свет | Вечерний Салют | «Салют» — другая традиция |

**Вердикт:** Не ошибка, а два набора терминов. Книга использует описательные, рубрикатор — краткие. Но в бэкенде нужно хранить ОБА набора имён.

### 3.2. Пропущенный контекст: Тайный/Явный союз и Война/Мир

В рубрикаторе есть отдельная строка без заголовка:

| 0 | 3 | 6 | 9 |
|---|---|---|---|
| Тайный союз | Война | Явный союз | Мир |

Это **дополнительная характеристика Опорного креста**, которую я не включил ни в один документ:
- 0 = Тайный союз (скрытое единение, ночь)
- 3 = Война (борьба на горизонте, восток)
- 6 = Явный союз (открытое единение, день)
- 9 = Мир (примирение, запад)

Пара 0↔6: Тайный↔Явный. Пара 3↔9: Война↔Мир. Это усиливает описание Опорного креста.

### 3.3. Чаша Света / Чаша Тьмы — неточность

В LOGIC_SPEC написано:
```
Верхняя половина (3→4→5→6→7→8→9) = Чаша Света
```

Книга (п.306): «Пять дневных, пять ночных и **ещё два**, соединяющих все времена дневные со всеми временами ночными».

Позиции 3 и 9 — **не в чашах**, а между ними. Они соединяют чаши.

**Корректировка:**
```
Чаша Света = {4, 5, 6, 7, 8} — пять дневных
Чаша Тьмы  = {10, 11, 0, 1, 2} — пять ночных
Соединения = {3, 9} — два времени горизонта
```

### 3.4. Кривая освещённости — отсутствует оговорка

LIGHT_CURVE = [0, 0.08, 0.25, 0.50, 0.75, 0.92, 1.00, 0.92, 0.75, 0.50, 0.25, 0.08]

Книга НЕ даёт числовых значений. Книга описывает КАЧЕСТВЕННОЕ поведение: ускорение в 2 и 8, замедление в 4 и 10, максимальную скорость в 3 и 9, нулевую в 0 и 6.

Симметричная линейная интерполяция — это **допущение**, которое **не учитывает** ускорение/замедление (п.262–271). Реальная кривая ближе к синусоиде.

**Корректировка:** Добавить примечание: «Приблизительные значения. Книга описывает нелинейную кривую с ускорением в позициях 2,8 и замедлением в 4,10.»

### 3.5. Формула halfOppositionStruggle в бэкенд-коде

В LOGIC_SPEC есть функция `mirrorAcrossAxis(index, [3, 9])`, но она **не определена**. Нужна реализация:

```javascript
function mirrorAcrossAxis(index, axisPositions) {
  // Reflection map for axis 3-9 (horizontal):
  const mapStruggle = {0:6, 1:5, 2:4, 3:3, 4:2, 5:1, 6:0, 7:11, 8:10, 9:9, 10:8, 11:7};
  // Reflection map for axis 0-6 (vertical):
  const mapUnity = {0:0, 1:11, 2:10, 3:9, 4:8, 5:7, 6:6, 7:5, 8:4, 9:3, 10:2, 11:1};
  
  if (axisPositions[0]===3) return mapStruggle[index];
  if (axisPositions[0]===0) return mapUnity[index];
}
```

**Верификация по книге:**
- п.233 (ось 3–9): Сумерки(10)↔Спуск(8), КонецСвета(11)↔НачалоТьмы(7), Тьма(0)↔Свет(6), ПервыйСвет(1)↔ПоследняяТьма(5), Заря(2)↔Подъём(4) → ✅ карта верна
- п.253 (ось 0–6): ПервыйСвет(1)↔ПоследнийСвет(11), Заря(2)↔Сумерки(10), Проявл.Света(3)↔Проявл.Тьмы(9), Подъём(4)↔Спуск(8), ПоследняяТьма(5)↔ПерваяТьма(7) → ✅ карта верна

### 3.6. Формула heatRole — некорректна

В бэкенд-коде LOGIC_SPEC:
```javascript
heatRole: ['исток','нагревание','пик','остывание','конец'][...]
```

Это не учитывает, что роль зависит от конкретной дуги, и третья дуга «странная» (п.669). Формулу нужно переписать.

## 4. ОШИБКИ В CHECKLIST: ЛИНГВИСТИЧЕСКИЕ СВЯЗИ

### 4.1. Полочка 9 — неверные связи

Написано: «Дракон → порядок → строй → весы (баланс)»

Должно быть: «Птицы → Голубятник → Весы ♎ → оценка». Дракон стоит на позиции 10, не 9.

Корректная связь для 9: Птицы → голуби → Голубятник → Весы ♎ → взвешивание → Бухгалтерия → Царская ложа (театр) → Кровать (спальня) → Заход Солнца.

### 4.2. Полочка 10 — неверные связи

Написано: «Скорпион ♏ → скрытность → сумерки → кабинет»

Должно дополняться: Дракон (животные) → «пчёлы — дракон двора» → Пчёлы/Мёд (постройки) → Скорпион ♏ → жалит → Юр. отдел (жалящие вопросы) → Дирекция (театр) → Сумерки.

### 5.2. Праны — согласованы ✅

CHECKLIST: Уровень 2 использует ту же формулу.
Треугольники стихий совпадают.

### 5.3. Противостояния — согласованы ✅

LOGIC_SPEC: opposition(i) = (i+6)%12
CHECKLIST: Уровень 4 использует ту же формулу, список пар совпадает.

### 5.4. Физиология — частично согласована ⚠️

LOGIC_SPEC описывает Лёгкие/Тяжёлые/Ожидательные времена (п.273–277).
CHECKLIST **не содержит** этой проверки как отдельного уровня. Рекомендуется добавить **Уровень 8: Проверка физиологии** — ускоряется/замедляется/постоянно.

### 5.5. Чаши Света/Тьмы — не согласованы ⚠️

LOGIC_SPEC: описывает чаши как 3→...→9 и 9→...→3

Несогласованность: CHECKLIST правильно исключает 3 и 9 из половин (как «горизонт»), а LOGIC_SPEC включает их. Нужно привести LOGIC_SPEC к версии CHECKLIST.

### 5.6. Закон «накопление → переход» — не отражён в чек-листе ⚠️

LOGIC_SPEC (п.286–298): каждое долгое = накопление, каждое короткое = переход в новое качество.
CHECKLIST Уровень 5 (Соседство) частично покрывает, но не формулирует явно.

**Рекомендация:** Добавить в Уровень 5 явный вопрос: «Что накапливается в предыдущем долгом времени? Во что это переходит в данном коротком?»
