# Контентное дерево «Ясна Суток» · спецификация v1.1

> **Changelog v1.1:** добавлены 3 типа атомов (`number`, `process`, `opposition`); `meta.see_also` мигрирует в структурированное `meta.links` с 5 типами связей; добавлен раздел «Реестр категорий по темам»; добавлен раздел `rejected` для отброшенных пунктов.

## Цель

Превратить 944 пункта книги в **машиночитаемый контентный фундамент** для геймифицированного продукта (квизы, задачи, иллюстрации). Книга — мастер-данные. Вся терминология и формулировки идут дословными цитатами с привязкой к пункту и PDF-странице. Никакой отсебятины: если факта в книге нет — его нет в дереве.

## Источник истины

| Слой | Что это | Где |
|---|---|---|
| **L0 · PDF** | Скан книги, 105 разворотов, 944 пункта | `uploads/ясна суток.pdf` |
| **L1 · Конспекты узлов** | 11 узлов с дословными цитатами `[№N]` и привязкой `PDF p.X` | `Сутки/02_узел_01…11_узел_10*.md` |
| **L2 · Контентное дерево** | Атомизированные блоки + готовые вопросы для квизов | `Сутки/КОНТЕНТНОЕ_ДЕРЕВО/*.json` |
| **L3 · Геймифицированный продукт** | `trivia-bank.js`, движок, UI | `docs/preview/games/duel/` |

Каждый слой ссылается на предыдущий. Атом в L2 содержит `source.items` (номера пунктов) и `source.pdf_page`, что позволяет в одно действие проверить любую цитату.

## Иерархия

```
Книга «Ясна Суток»
└── Тема (Theme) ── 1 узел = 1 тема, 10 тем
    └── Категория (Category) ── смысловой подраздел темы (3-7 категорий на тему)
        ├── Атом (Atom) ── один факт / термин / список / правило
        ├── Иллюстрация (Illustration) ── чертёж / схема / визуальный приём
        └── Вопрос (Question) ── готовый игровой блок, ссылается на атом(ы)
```

## Структура атома

```yaml
atom:
  id:         T1.K1.A1                      # стабильный
  kind:       definition | fact | list | process | number |
              etymology | pairing | opposition | postulate | name | rule | scheme
  term:       "Ясна"                        # ключевое слово (если есть)
  quote:      "Ясна есть способ описания природы."   # ДОСЛОВНО из книги
  source:
    items:    [1]                           # список номеров пунктов
    pdf_page: 5
  meta:
    synonyms: ["Веда", "Гимн"]              # альтернативные имена внутри книги
    is_keystone: false                      # ключевой постулат темы (если true)
    notes:    null                          # пояснение составителя (не выводится игроку)
    links:                                  # типизированные связи между атомами
      see_also:     ["T1.K1.A8"]            # просто связано
      prerequisite: []                      # нужно знать ДО этого атома
      leads_to:     ["T2.K1.A1"]            # развивается дальше в этом
      contrasts:    []                      # противоположно этому (для opposition)
      applies_in:   []                      # применяется в этом узле/теме
  supports_questions:
    - single-choice
    - fill-blank
    - true-false
```

> Все поля в `meta.links` — **опциональны**, заполняются по мере обнаружения связи в книге. Главное — резерв имён, чтобы не мигрировать 300+ атомов на новую schema позже.

### Виды атомов (`kind`) — 12 типов

| Код | Что | Пример | Дополнительные поля |
|---|---|---|---|
| `definition` | Определение термина | «Ясна есть способ описания природы» (№1) | `term` |
| `fact` | Утверждение | «Ясну удобно начать изучать с световых суток» (№39) | — |
| `list` | Несортированный список с фиксированным числом элементов | 4 стихии · 4 праны · 5 чаш | `items_list[]` |
| `process` | **Упорядоченная последовательность** | восход → полдень → закат → полночь | `sequence[]`, `is_cyclic` |
| `number` | **Числовая величина с единицей** (позвоночник модели) | 12 учебников · 24 часа · 6 долгих времён · 4 риски | `value`, `unit`, `defines_term?` |
| `etymology` | Происхождение / родственные слова | «Ясна» ↔ «ясный, ясность» (№2) | `related_words[]` |
| `pairing` | **Связь** двух понятий — отличаются, но не противостоят | «знать ↔ ведать» (два слоя обладания) | `term_a`, `term_b`, `kind_a`, `kind_b` |
| `opposition` | **Антитеза** — взаимоисключающие | «свет ↔ тьма», «верх ↔ низ» | `term_a`, `term_b` |
| `postulate` | Аксиома, формула | «Закон одной вещи = закон любой» | `is_keystone?` |
| `name` | Альтернативное имя того же | Ясна = «Всеобщий Структурный Закон Мира» (№8) | `alt_name` |
| `rule` | Правило применения | «Изучать удобно с того, что легко наблюдать» | — |
| `scheme` | Геометрическая структура | Опорный Крест · Линия Борьбы | `references_illustration?` |

> **`pairing` vs `opposition`** — критическое различие. `pairing` означает «эти два связаны и различимы» (знать/ведать — два уровня одного отношения к вещи). `opposition` означает «эти два — антиподы, одно есть отрицание другого» (свет/тьма). В Ясне антипод-логика всюду, и движок генерации «найди антипод» опирается именно на `opposition`.

### Зарезервированные виды (на будущее)

`metaphor` (образное сравнение), `principle` (принцип организации, отличается от постулата), `example` (конкретный кейс применения). Включаются в спеку только когда встретятся в книге — иначе спека раздувается без пользы.

## Структура вопроса

```yaml
question:
  id:          T1.K1.Q1
  type:        single-choice | multi-choice | fill-blank | true-false | match-pair | order
  difficulty:  easy | medium | hard
  stem:        "Что такое Ясна?"
  correct:     "Способ описания природы"     # верный ответ (str или массив для multi/match/order)
  correct_alternatives: ["..."]              # для fill-blank: синонимы, которые движок принимает (опционально)
  options:
    - "Способ описания природы"              # верный
    - "Древнее искусство гадания"            # дистрактор
    - "Свод гимнов греческих богов"          # дистрактор
    - "Раздел физики XX века"                # дистрактор
  explanation:
    quote:    "Ясна есть способ описания природы. (№1)"
    source:   { items: [1], pdf_page: 5 }
  source_atoms: ["T1.K1.A1"]                 # каким атомам соответствует
  tags:        ["definition", "core"]
```

### `correct_alternatives` для fill-blank

Если для пробельного вопроса валидны несколько формулировок (синонимы, грамматические варианты), их перечисляют в `correct_alternatives`. Движок игры нормализует ввод (lowercase, удаление лишних пробелов) и проверяет на вхождение в `[correct, ...correct_alternatives]`.

```yaml
type:        fill-blank
correct:     "не может"
correct_alternatives: ["не способно", "нельзя"]   # принимаются как верные
```

### `match-pair` — правило однозначности

В match-pair каждое слово в любой колонке должно быть **однозначным** в контексте всей пары. Если слово используется в книге в разных смыслах (например, «Веда» — и то, что есть, и то, как её называет наука), match-pair не подходит — нужно single-choice или fill-blank. Квалификаторы в скобках («Веда (для науки)») в качестве костыля **запрещены** — они выглядят как разработческий артефакт.

## Реестр отброшенных пунктов (`rejected`)

Не каждый из 944 пунктов даёт атом. Часть пунктов — повторы, рекапы, мета-комментарии, переходные анонсы или нечитаемые из-за скана. Чтобы был полный audit-trail (никто не сможет сказать «вы пропустили пункт N»), каждая тема ведёт массив `rejected`:

```yaml
rejected:
  - { item: 5,  reason: "duplicate of #4" }
  - { item: 20, reason: "meta-commentary on word 'народ', no testable content" }
  - { item: 25, reason: "recap of items 7–15" }
  - { item: 36, reason: "unclear phrasing in source, requires clarification" }
  - { item: 41, reason: "transitional teaser to T2" }
```

**Метрика покрытия темы:**
```
coverage = atoms_count / (atoms_count + rejected_count)
```

| Уровень | Что значит |
|---|---|
| **≥ 90%** | Узел плотный, все пункты несут факты. Нормально для коротких узлов (T1, T3, T10). |
| **70–90%** | Здоровый уровень. Часть пунктов — повторы или рекапы, что естественно. |
| **60–70%** | Тёплый. Скорее всего автор делает много мета-замечаний. Допустимо, но стоит проверить. |
| **< 60%** | Тревожный сигнал. Либо тема плохо атомизирована, либо книга в этом узле слабая. Звать ревьюера. |

## Принципы дистракторов (правдоподобные неверные ответы)

1. **Из той же темы** — лучше всего: альтернатива из того же списка книги (другой риск, другая стихия).
2. **Семантические соседи** — близкие по смыслу понятия (например, «закон» вместо «способ»).
3. **Известные ложные сопоставления** — то, чем Ясна **не** является («раздел физики», «искусство гадания»), — берётся только если книга явно отрицает (типа №9, 17).
4. **НИКАКИХ выдуманных терминов** — все альтернативы либо из книги, либо общеизвестные.

## Пайплайн «атомы → вопросы»

Каждый атом помечен полем `supports_questions`. При генерации:

| Kind атома | Подходящие типы вопросов |
|---|---|
| `definition` | single-choice, fill-blank, true-false |
| `fact` | true-false, single-choice |
| `list` | multi-choice (выбрать N из 8), order, single-choice (счёт элементов) |
| `etymology` | single-choice (от какого слова), match-pair |
| `pairing` | match-pair, single-choice |
| `postulate` | fill-blank, true-false |
| `name` | match-pair, single-choice (синоним) |
| `rule` | true-false, single-choice |
| `scheme` | картинка-вопрос, match-pair |

Часть вопросов пишется вручную (для контроля качества), часть — авто-генерируется по правилам.

## Стабильность ID

Раз присвоенный ID атома никогда не меняется. Если атом удаляется — его слот резервируется (помечается `deprecated: true`). Это критично, т.к. на ID атомов будут ссылаться:
- Вопросы (по `source_atoms`)
- Аналитика игр (какой атом завалили, какой пройден)
- Карты освоения (которые в UI показываются процентом)

## Чек-лист одного узла (готовности темы)

- [ ] Все пункты узла прочитаны и атомизированы или явно отброшены (мета-комментарии, повторы)
- [ ] Каждый атом имеет цитату дословно из книги
- [ ] Каждый атом имеет `source.items` и `source.pdf_page`
- [ ] Категории определены, атомы распределены без пропусков
- [ ] Минимум 5 готовых вопросов на тему
- [ ] Дистракторы — все правдоподобные, без отсебятины
- [ ] Иллюстрации (если есть в узле) внесены отдельным списком

## Версионирование

`version` на уровне файла: `1.0.0`. Менять при:
- patch → правка цитат, опечаток
- minor → добавление новых атомов / вопросов
- major → переименование ID или изменение схемы (избегать)
