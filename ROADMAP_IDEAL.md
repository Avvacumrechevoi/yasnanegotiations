# Роадмап «Идеальная Ясна» — план доработок

> Источник: глубокий аудит 10.06.2026 (112 находок) + продуктовый разбор.
> Документ ведётся как живой план. Отметки: ⬜ не начато · 🟡 в работе · ✅ сделано.

## 0. Принципы

1. **Один сквозной продукт, а не набор миниигр.** Конструктор ↔ Обучение ↔ Игра ↔ Переговоры ↔ Рейтинг должны вести друг к другу в логичных местах, а не только через верхний бар.
2. **Не обещать того, чего нет.** Любая кнопка/CTA либо работает, либо честно помечена «скоро» без засчитывания прогресса.
3. **Прод-ветка main = автодеплой.** Любая правка едет через feature-ветку и PR; main защищаем required-check'ом (см. Волну 5).
4. **Каждый под-продукт изолирован** — своя страница, свой namespace в localStorage, свой контент-файл. Можно развивать независимо, не трогая остальное.

```
start.html (лендинг) ─┬─ index.html  Конструктор (React SPA, 12-полочная звезда)

### Целевая (добавляем изолированный раздел «Переговоры»)
```
start.html ─┬─ index.html         Конструктор
            ├─ learn.html         Обучение
            ├─ duel.html          Игра (теория Ясны)
            ├─ negotiations.html  ⭐ НОВОЕ: Тренажёр переговоров (прикладной под-продукт)
            └─ rating.html        Рейтинг (+ живой топ игроков)

«Переговоры» — отдельный прикладной тренажёр на базе модели «Ясна Переговоров»
(12 стадий, уже заведена в конструкторе: `docs/core/data.js:76`). Живёт изолированно:
`docs/negotiations.html` + `docs/negotiations/*`. См. раздел 7.

## 8. ⭐ Изолированный под-продукт: «Тренажёр переговоров»

### 8.1. Зачем
Имя репозитория — *yasnanegotiations*. Переговоры — прикладное ядро бренда.
Модель «Ясна Переговоров» уже формализована (12 стадий цикла А↔В). Превращаем её
из абстрактного пресета звезды в **практический тренажёр**: пользователь учится
видеть стадию переговоров и вести её осознанно.

### 8.2. Модель (12 стадий, из `data.js`)
Игра А↔В (резонанс/десонанс): нарастание контакта → пик → спад через понимание или срыв.
```
 1 Информационное Поле     7 Противостояние А↔В
 2 Хочу / Не Хочу          8 Обоюдное Понимание
 3 Привлечь и Заинтересовать 9 Недопонимание
 4 Открытие Позиции       10 Десонанс / Срыв
 5 Резонирование          11 Точечный Удар
 6 Ограничения и Контр    12 Итог: Успех / Неудача
```

### 8.3. Архитектура (изоляция)
```
docs/negotiations.html        — точка входа (standalone, VK-dark токены, SiteNav)
docs/negotiations/
  trainer.js                  — логика тренажёра (vanilla, без сборки)
  trainer.css                 — стили раздела
  scenarios.js                — КОНТЕНТ: банк реплик/ходов с привязкой к стадии
```
- Namespace localStorage: `yasna_neg_*` (не пересекается с конструктором/дуэлью).
- Контент отделён от логики (`scenarios.js`) — расширяется без правки кода.
- Никаких зависимостей от бандла конструктора → развивается независимо.

### 8.4. Дорожная карта самого тренажёра
- **MVP (этот PR) ✅**: интерактивная карта 12 стадий + дрилл «Определи стадию»
  (показываем реплику/ход → игрок выбирает стадию → разбор). Прогресс в localStorage.
- **v0.2 ⬜**: сценарии-кейсы (5–7 реплик подряд = одна сделка), разбор по стадиям в конце.
- **v0.3 ⬜**: «свой» режим — разложить свои переговоры по 12 полкам (как мини-конструктор).
- **v0.4 ⬜**: роли (Покупатель/Продавец, Жёсткий/Мягкий оппонент), ветвление.
- **v0.5 ⬜**: связка с дуэлью (категория «Переговоры») и аккаунтом (общий прогресс).

### 8.5. Контент-бэклог
Наполнить `scenarios.js` 40–60 репликами (по 3–5 на стадию) из практики переговоров,
с коротким разбором «почему это именно эта стадия». Источник — корпус книги +
переговорные кейсы. Сейчас заложен стартовый банк (~24 реплики) для проверки механики.
