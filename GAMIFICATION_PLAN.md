# Геймификация yasnanegotiations — план и каталог игр

Дата: 2026-05-07. Статус: проектирование, готово к реализации.

## 0. Зачем игры

Существующие модули решают разные задачи обучения:

| Модуль | Что делает |
|---|---|
| **Гид** | Линейный сторителлинг, проводит через Ясну за один сеанс |
| **Уроки** | Пошаговые лекции с примерами и проверками понимания |
| **Проверка** | Аналитический инструмент — оценить готовность собственной Ясны |
| **Игры** ★ NEW | **Активный recall** — заставить мозг доставать знание из памяти, а не узнавать его при показе |

Психологически это разные режимы: чтение тура — пассивный recognition; игра — активный recall. Без recall знание не закрепляется. Игры закрывают эту дыру.

## 1. Архитектурные принципы

Принципы те же, что у тур/уроков — модуль изолирован, не ломает остальное:

```
docs/preview/games/
├── engine.js           — GameRunner: общий рантайм (счёт, таймер, подсказки, прогресс)
├── games-index.js      — реестр + порядок появления в каталоге
├── progress.js         — localStorage namespace «yasna_games_progress»
└── game-XX-name.js     — каждая игра отдельным файлом, регистрируется в реестре
```

```
docs/preview/styles/
└── games.css           — отдельный namespace .game-* для стилей игр (опционально)
```

**Регистрация:**
```js
window.YasnaGames.register('match-cross', {
  id: 'match-cross',
  category: 'foundation',          // foundation/composition/speed/puzzle/meta
  difficulty: 1,                   // 1..5
  estimatedMinutes: 3,
  title: 'Найди Опорный Крест',
  description: '...',
  prerequisites: ['lesson-2-four-pillars'],  // что должен сначала пройти
  prereqGames: [],                 // что из игр должен сначала пройти
  yasnaIds: ['sutok'],             // на каких Яснах работает (или ['*'] — на любой)
  dataDeps: ['T','CR','PR'],       // что из window.YasnaData нужно
  init: ({ yy, ui, score, onWin, onLose, onHint }) => { ... },
  render: ({ ... }) => <ReactComponent />
});
```

**Engine берёт на себя:**
- запуск/перезапуск, таймер
- подсчёт очков (правильно/неправильно/штраф за подсказку)
- хранение прогресса в `localStorage['yasna_games_progress']` (отдельный namespace, никаких миграций для существующих данных)
- общий UI (header с названием игры, прогресс-бар, кнопка «Сначала», подсказки)
- расшаривание ссылки на конкретную игру `#game=XXX`

**Что НЕ переиспользуем:** свой engine, свои стили, свой namespace в localStorage. Никаких изменений в `app.js`, `core/*`, `tours/*`, `lessons/*` — кроме одной кнопки «Игры» в шапке (как уже есть «Уроки/Гид/Стихии/Проверка»).

**Зависимости (только чтение, не пишут):**
- `window.YasnaData.{T, FL, CR, PR, COMP, GLOSS, POS_DESC, gc, gp, opp}`
- `window.YasnaCore.Star` (для визуала диаграммы внутри игры)

## 2. Каталог игр (15 концепций)

Сгруппированы от простых к сложным. Каждая привязана к конкретной механике/закону Ясны — это не «накладной» геймплей, а упражнение, которое нельзя выполнить без понимания закона.

### Категория A — Фундамент (узнавание + отображение, ⏱ 2–4 мин)

#### A1. Найди Опорный Крест
- **Геймплей:** диаграмма Ясны Суток с 12 полками. Игрок кликает на 4 полки, считая их опорными. Подсветка зелёным/красным.
- **Закон:** Опорный Крест = 0/3/6/9 (Ночь, Проявление Света, День, Проявление Тьмы).
- **Усложнения:** на больших уровнях — Ясна без подписей (только цифры), позже — без цифр.
- **Скоринг:** 4 правильных → 100; за 1 ошибку −25, за 1 подсказку −15.
- **Триггер мастерства:** 3 раза подряд 100/100 → разблокирует **A2**.

#### A2. Найди Крест Управления
- То же самое, но Крест Управления (1/4/7/10).
- После прохождения A1+A2+A3 (Крест Веры) разблокирует **B1**.

#### A3. Найди Крест Веры
- Крест Веры (2/5/8/11).

#### A4. Антиподы
- Игроку показывается одна полка («Утро»). Надо ткнуть в её антипод. 6 раундов на разные стартовые полки.
- **Закон:** `opp(i) = (i+6)%12`.
- **Усложнение:** Ясна Года, Жизни — антипод не очевидный, надо знать материал.

#### A5. Назови позицию
- Звучит/показывается имя полки («Восход»). Игрок жмёт цифру 0–11.
- **Закон:** имена-метки → числовая адресация.
- 12 раундов, время 8с/раунд.

### Категория B — Композиция (построение, ⏱ 5–10 мин)

#### B1. Заполни пустую Ясну
- Дан *шаблон* (только метки 0/3/6/9 — Опорный Крест Суток). Игроку даётся пул 8 правильных + 4 «отвлекающих» (с похожих Ясн). Перетаскивает на 1/2/4/5/7/8/10/11.
- **Закон:** позиции имеют физическое и временное место в цикле; нельзя переставить местами.
- **Скоринг:** 8 правильных → 100; путаница соседей (позиции 2 и 4) — мягче (−10), путаница противоположных (2 и 8) — жёстче (−30).

#### B2. Разорванный цикл
- Дан случайный порядок 12 полок. Игрок выстраивает в правильную последовательность через drag-and-drop.
- 3 уровня: Сутки → Год → Жизнь.

#### B3. Композитор Праны
- Дан Тригонник (3 полки одной праны: 0,4,8 — Земля). Игрок должен подобрать «общую черту» из 4 предложенных.
- **Закон:** позиции одной праны разделяют качество (стабильность для Земли, спад для Воды, и т.д.).
- 12 раундов (4 праны × 3 шаблона).

#### B4. Свой день в Ясне Суток
- Игрок переносит свои реальные события (вводит сам или выбирает из шаблонов «обед», «зарядка», «совещание») на ближайшую полку Суток. Алгоритм даёт обратную связь — насколько корректно с т.з. позиций.
- **Учебная цель:** перенос абстрактного знания в личную жизнь.

### Категория C — Скорость (флэш-карты, ⏱ 1–3 мин)

#### C1. Молния — Кресты
- 30 секунд. На экране одна полка («Заря»). Игрок жмёт «Опора / Управление / Вера». Сколько успеет.
- Используется для разогрева перед уроком.

#### C2. Молния — Праны
- То же, но «Земля / Вода / Воздух / Огонь» для каждой полки.

#### C3. Молния — Долгое/Короткое
- «Длинное» (чётные) или «Короткое» (нечётные).

#### C4. Молния — Антиподы
- Показывается полка, надо за 3 секунды ввести (через 12 кнопок) её антипод.

### Категория D — Головоломки (логика, ⏱ 8–20 мин)

#### D1. Ясна² Sudoku
- Поле 12×12 (drill-down), часть ячеек заполнена. Игроку дан Шаблон верхней Ясны. Заполнить пустые ячейки так, чтобы каждая колонка-полка осмысленно вкладывала в свою «материнскую» полку.
- Подсказки: «эту колонку определяет полка 4 → её природа: подъём, культура».
- Авто-проверка через эвристики (ключевые слова, glossary).

#### D2. Найди Ошибку
- Дана Ясна с одной случайной ошибкой (антипод поменян, прана не та, опорная позиция перепутана). Игрок находит ошибку за лимит времени.
- Использует существующую Verification как back-end-чек.

#### D3. Лента Мёбиуса
- Игроку показан Зодиакальный круг с 12 Мифами. Надо «провести нить» от Овна → Тельца → … так, чтобы получилась лента (хвост в рот, перекрут на 0/6).
- Драг-рисование линий.

### Категория E — Меta-игра (прогресс, ⏱ долгая)

#### E1. Карта Ясн
- 44 шаблона = 44 ячейки. Каждая ячейка — мини-игра «пройди A1+A2+A3 на этой Ясне». Получаешь звезду.
- Долгосрочная цель: собрать все 44 звезды.

#### E2. Ежедневный челлендж
- Каждый день рандомизируется одна игра из A/B/C на одну случайную Ясну. Streak (серия дней подряд).
- Минимально, без push-уведомлений (это статичный сайт). Просто отметка в localStorage.

## 3. Скоринг и прогресс

### Per-game очки
- 100 очков за идеально (без ошибок и подсказок)
- −25 за каждую серьёзную ошибку
- −10 за «мягкую» (соседи)
- −15 за подсказку
- Минимум 0, не отрицательное

### Прогресс-метрики (localStorage)
```js
yasna_games_progress = {
  version: 1,
  byGame: {
    'find-cross': { plays: 12, bestScore: 100, lastScore: 85, streak: 3, mastered: true },
    ...
  },
  daily: { lastDay: '2026-05-07', streak: 5 },
  totalStars: 18,
}
```

### Звёзды
- 0/3/6/9 ⭐ за score < 60 / 60–79 / 80–99 / 100
- Звёзды накапливаются по игре, по Ясне, итого

## 5. Стиль и UX

- Тот же design language что в гиде — тёмный фон `#0e1019`, `#d4a574` accent
- Шрифты от существующего CSS (var(--sans), var(--serif))
- Анимации не громче существующих (`fadeUp` 0.55s — переиспользуем)

## 7. Риски и не-цели

**Риски:**
- *Перегрузка UI* — 15 игр в одном меню. Решение: каталог с категориями + рекомендация дня.
- *Сложность A4 на Ясне Жизни* — антиподы там неочевидны. Решение: отдельный путь сложности — easy (Сутки), medium (Год), hard (Жизнь).
- *localStorage квота* — у пользователя могут уже быть данные `yasna2_subdata`. Решение: отдельный namespace `yasna_games_progress`, не больше 50KB.

**Не-цели:**
- ❌ Не делать соревновательный мультиплеер — это статичный сайт, без сервера.
- ❌ Не делать аудио и музыку — портит UX в офисе/публичном месте.
- ❌ Не делать «pay to win» / lootbox-механики — это образовательный продукт.
- ❌ Не делать сложные RPG / нарративные движки — есть Гид и Уроки для этого.
- ❌ Не дублировать функциональность Verification — Verification остаётся аналитическим инструментом, игры — тренажёр.

## 10. Метрики успеха

После запуска P0–P3 можно понять, попадаем ли:

- **DAU/MAU самой геймификации** — сколько % посетителей сайта запустили хоть одну игру
- **Среднее количество попыток до mastery** на A1 — должно сходиться к 5–8 (если выше — игра слишком сложная или объяснение слабое)
- **Drop-off rate на B1** — сколько % бросают на drag-and-drop (показывает usability)
- **Streak длиннее 7 дней** — сколько % пользователей возвращаются ежедневно (главный показатель retention)

Метрики собирать невозможно (нет аналитики на статике), поэтому ориентируемся на качественные сигналы — фидбек пользователей, скриншоты, ручную перепроверку UX.

## Резюме

15 игр в 5 категориях, изолированный модуль `docs/preview/games/`, не ломает ничего. Старт — простая A1 на пол-дня. Остальное — 5 фаз × 1 день каждая. После P3 уже играбельная подсистема, после P5 — полноценный продукт.

**Когда давать команду «делай P0»:** когда появятся 4–6 свободных часов и желание увидеть, как работает activerecall на Ясне.

**Сначала делать P0.** Если зайдёт — катиться дальше по фазам. Если не зайдёт — просто оставить P0 как proof-of-concept без потери для остального продукта.
