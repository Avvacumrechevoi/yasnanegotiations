# Система рейтинга Ясны · v2

Документ описывает **две параллельные шкалы** прогресса игрока: индивидуальное Освоение знания и Турнирный Ранг (PvP).

## Зачем две шкалы

Раньше была одна общая «бусины + ступень» — она смешивала:
- игру с лёгкой Тенью-ботом (где побеждать просто)
- игру с живым соперником своего уровня

Это плохо по двум причинам:
1. **Несправедливо**: 100 партий со слабым ботом = 100 партий против сильных игроков. Соревновательной правды нет.
2. **Демотивирует исследование**: страх «упасть» в ранге заставляет избегать сложных PvP. Прогресс в знании смешан с риском в состязании.

Решение — разделить:
- **Освоение** — личный рост, накопительный, не падает. Метрика «сколько знаешь».
- **Каста (PvP)** — соревновательный, ELO-подобный. Метрика «как играешь против людей».

Игрок видит обе шкалы и сам выбирает, какую качать.

## Tier 1 — Освоение

> **Что это**: твой личный путь по Ясне. Сколько знаешь, как точно отвечаешь, как часто играешь.

**Валюта**: ✦ **Бусины** (единая для соло и PvP).

**За что начисляются** (на каждый верный ответ):
- **Базовая**: 10 ✦
- **Скоростной бонус**: до +5 ✦ (за быстрые ответы; чем быстрее, тем больше)
- **Стрик-множитель**: ×1.2 (3 верных подряд) · ×1.5 (5) · ×2.0 (7+)

**Никогда не теряются.** Бусины — это память о пройденном пути. Только растут.

**Соло vs PvP**:
- В соло (с Тенью) — стандартный расчёт
- В PvP — те же бусины, но играть труднее → объективно растёт быстрее

**Уровни (Ступени)**:

| Ступень | Бусины (порог) | Подуровни |
|---|---|---|
| **Послушник** | 0 — 999 | I — IX (каждые 100 ✦) |
| **Студент** | 1 000 — 4 999 | I — IX (каждые ≈444 ✦) |
| **Бакалавр** | 5 000 — 14 999 | I — IX (каждые ≈1 111 ✦) |
| **Магистр** | 15 000 — 49 999 | I — IX (каждые ≈3 888 ✦) |
| **Доктор** | 50 000 + | I — ∞ (каждые 5 000 ✦) |

> Названия идут из академической иерархии: Послушник → Студент → Бакалавр → Магистр → Доктор. «Доктор» — высшая учёная ступень, открытая, без потолка.

**Что показывается**: «Послушник VI · 591 ✦ · до Студента 409 ✦» (текущий и до следующей ступени).

## Tier 2 — Каста (PvP)

> **Что это**: твоё соревновательное место среди игроков. Как ты играешь против людей — точнее, быстрее, расчётливее их.

**Валюта**: **PR** (Player Rank), числовой Glicko-2-подобный рейтинг. Старт = 1200.

**За что меняется** (только в PvP, БОТ НЕ ВЛИЯЕТ):
- Победа: **+ΔPR** (зависит от соперника)
- Поражение: **−ΔPR**
- Ничья: малое ΔPR

ΔPR считается по упрощённой ELO-формуле:
```
ΔPR = K × (S − E)
S = 1 (победа), 0.5 (ничья), 0 (поражение)
E = 1 / (1 + 10^((opp_rank − my_rank) / 400))
K = 32 (стандартный коэффициент)
```

Пример: PR 1200 побеждает PR 1400 → ΔPR ≈ +24. Проигрывает → −8.

**Уровни (Каста)**:

| Каста | PR | Описание |
|---|---|---|
| **Странник** | < 1100 | Только начал PvP |
| **Знаток** | 1100 — 1399 | Регулярный игрок |
| **Игрок** | 1400 — 1699 | Опытный соперник |
| **Магистр Игры** | 1700 — 1999 | Сильный игрок |
| **Великий Магистр** | 2000 + | Вершина (Magister Ludi у Hesse) |

> Hesse: «Магистр Игры» (Magister Ludi) — глава ордена в «Игре в бисер». В Касталии один такой за поколение. У нас «Великий Магистр» как высшая каста — реверанс к этой иерархии.

**Что показывается**: «Игрок · PR 1542» (название касты + точное число).

**Можно ли упасть**: да, при поражениях PR снижается. Это здоровый соревновательный сигнал — побуждает играть осторожно с сильными.

**Защита от деморализации**:
- Floor 800: ниже не упадёшь
- Decay только если играешь против явно более сильных (что — твой выбор)
- Caste никогда не «обнуляется», только в пределах диапазона

## Связь между тиерами

| Действие | Освоение (✦) | Каста (PR) |
|---|---|---|
| Соло-партия (Тень) | + бусины | — не меняется |
| PvP-победа | + бусины | + ΔPR |
| PvP-поражение | + бусины (за верные ответы) | − ΔPR |
| PvP-сдача | + бусины (за то что было) | − ΔPR (penalty) |
| Дисконнект | НЕ начисляется ничего | НЕ меняется |

**Принцип**: Бусины → выучил, и это с тобой. Каста → выиграл против равного, и это сейчас.

## Технические детали

### Хранение

**Локально (localStorage)**:
- `profile.busey` — общее число бусин (накопление)
- `profile.pr` — текущий PR (умолчание 1200)
- `profile.castName` — кеш имени касты (вычисляется из PR)
- `profile.history.pvp[]` — история PvP-партий: { matchId, opponentPR, result, deltaPR, ts }

**На сервере (YDB `matches`)**:
- Новое поле `pr_change` Int32 — изменение PR (только для PvP-матчей)
- Новое поле `pr_after` Int32 — PR после матча
- Существующие `score`, `result` остаются

### Расчёт на клиенте

После завершения PvP-партии:
1. Из `transport` приходит `oppPR` (соперник передаёт свой текущий PR)
2. Локально считаем `deltaPR` по формуле
3. Обновляем `profile.pr += deltaPR`
4. Отправляем `submitMatch` с `pr_change` и `pr_after`

Соперник делает то же самое. Сходимость гарантирована (формула симметрична).

### UI-точки

1. **Профиль (хедер duel-page)**: показать обе шкалы
   - "Послушник VI · 591 ✦"
   - "Странник · PR 1200"

2. **VS-screen перед партией**: для PvP — показать обе касты соперников
   - "Dmitry · Послушник VI · PR 1542 (Игрок)"
   - "test2 · Студент II · PR 1380 (Знаток)"

3. **Финальный экран**: разделить виды наград
   - Solo: "Получено: +48 ✦"
   - PvP: "Получено: +48 ✦ · +24 рангу (PR 1224 → Знаток II)"

4. **Хроника**: сортировка по типам матчей (соло/PvP), показ ΔPR для PvP

5. **Партитура** — стандартная по темам

### Миграция

Существующие игроки с `profile.busey > 0`:
- Бусины остаются как есть
- PR начинается с 1200 (никто не наказан)
- На старте все «Странники» — поднимаются играя
